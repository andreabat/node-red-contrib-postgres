# Phase 2 — UI Design Contract

> Visual and interaction contract for the Node-RED editor panels of PostgresDBNode and PostgresNode.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | `none` — Node-RED native editor framework |
| Component library | Node-RED editor widgets: `RED.tabs`, `RED.editor`, `typedInput`, jQuery `form-row` |
| Icon library | Font Awesome 4 (`fa fa-*` classes, pre-existing in Node-RED) |
| Font | Inherit from Node-RED editor (no custom font declaration) |
| Code editor | CodeMirror 6 (replaces Ace), `@codemirror/lang-sql` PostgreSQL dialect |
| I18n framework | Node-RED `data-i18n` attributes + `locales/en-US/postgrestor.json` |

---

## Layout Conventions

All panels use Node-RED's standard `form-row` class for vertical stacking. No custom CSS layout — inherits Node-RED editor defaults.

| Token | Value | Usage |
|-------|-------|-------|
| form-row | Inherits Node-RED default vertical rhythm | Every config field is a `.form-row` |
| Tab container | `RED.tabs.create()` with UL + div children | PostgresDBNode multi-panel grouping |
| typedInput width | `style="width: 80%"` on editable fields | Consistent with existing host/port/database fields |
| Checkbox alignment | `display: inline-block; width: auto; vertical-align: top` | Toggle fields (type mapping, named params) |
| Editor height | `300px` min-height, `150px` min | SQL editor panel |
| Progressive disclosure | jQuery `.hide()` / `.show()` on field group containers | SSL cert fields, DATABASE_URL vs manual fields |

---

## Typography

Node-RED editor inherits system font. No custom font declarations needed.

| Role | Effect | Usage |
|------|--------|-------|
| Form labels | `<label>` with `data-i18n` + Font Awesome icon | All config fields |
| Tooltips | `[title]` attribute on labels (Node-RED standard) | Parameter explanations, field guidance |
| Tab labels | `RED.tabs` default styling | Config node tab navigation |
| Badge text | `node.status({ text: "..." })` — Node-RED status bar | Runtime pool health, query results |

---

## Color

### Node Type Colors (Node-RED palette and flow canvas)

| Node | Color | Usage |
|------|-------|-------|
| PostgresDBNode | `rgb(5, 93, 193)` (existing, unchanged) | Config node badge in flow canvas |
| PostgresNode | `rgb(5, 93, 193)` (existing, unchanged) | Flow node badge in flow canvas |

### Runtime Status Badge Colors

| State | Fill | Shape | Text Pattern |
|-------|------|-------|-------------|
| Pool healthy (active < 80% max) | `green` | `ring` | `Active: N  Idle: M  Waiting: W  Total: T` |
| Pool near limit (active ≥ 80% max) | `yellow` | `ring` | `Active: N  Idle: M  Waiting: W  Total: T ⚠` |
| Pool error / disconnected | `red` | `ring` | `DB: <error message>` |
| Query success | `green` | `ring` | `Query ok. N rows returned` |
| Query timeout | `yellow` | `ring` | `Query timeout after Nms` |
| Query error | `red` | `ring` | `Error: <sqlState> — <message>` |

### Status Badge Updates
- Pool health: polled every 30 seconds via `pool.totalCount / pool.idleCount / pool.waitingCount` (pg.Pool read-only properties)
- Query status: set immediately on result/error, cleared on node close with `node.status({})`

## Editor Panel Contracts

### PostgresDBNode — Config Panel

#### Tab 1: Connection (`postgrestor-config-tab-connection`)

| Field | Widget | Default | typedInput Types | Notes |
|-------|--------|---------|-----------------|-------|
| Name | `<input type="text">` | `""` | n/a | Existing, unchanged |
| Database Source | Radio buttons: `Manual` / `DATABASE_URL` | `Manual` | n/a | New: toggles between manual fields and URL field |
| Host | typedInput | `127.0.0.1` | `str, flow, global, env` | Existing. **Hidden when Database Source = DATABASE_URL** |
| Port | typedInput | `5432` | `num, flow, global, env` | Existing. **Hidden when Database Source = DATABASE_URL** |
| Database | typedInput | `postgres` | `str, flow, global, env` | Existing. **Hidden when Database Source = DATABASE_URL** |
| DATABASE_URL | typedInput | `""` | `str, flow, global, env` | **New.** Visible only when Database Source = DATABASE_URL. Placeholder: `postgresql://user:pass@host:5432/db` |
| SSL Mode | `<select>` dropdown | `disable` | n/a | **New.** Options: `disable`, `require`, `verify-ca`, `verify-full`. Replaces deprecated boolean SSL toggle. |
| SSL (deprecated) | `<input type="checkbox">` disabled, greyed out | `false` | n/a | **Migration field.** Only rendered if config had `ssl: true` before upgrade. Tooltip: "Migrated to sslmode: require. Use the SSL Mode dropdown below." |

**Progressive disclosure rules (D-01):**
- When `SSL Mode` is `verify-ca` or `verify-full`: show SSL credential fields (see Security tab)
- When `SSL Mode` is `disable` or `require`: hide SSL credential fields

**Database Source progressive disclosure:**
- When `Manual`: show Host, Port, Database; hide DATABASE_URL
- When `DATABASE_URL`: hide Host, Port, Database; show DATABASE_URL

#### Tab 2: Security (`postgrestor-config-tab-security`)

| Field | Widget | Default | typedInput Types | Notes |
|-------|--------|---------|-----------------|-------|
| User | `<input type="text">` | `""` | n/a (credential) | Existing, unchanged |
| Password | `<input type="password">` | `""` | n/a (credential) | Existing, unchanged |
| SSL CA Certificate | `<textarea>` (credential) | `""` | n/a (credential) | **New.** Visible only when SSL Mode ∈ {verify-ca, verify-full}. Stored in Node-RED encrypted credential storage (D-02). |
| SSL Client Certificate | `<textarea>` (credential) | `""` | n/a (credential) | **New.** Visible only when SSL Mode ∈ {verify-ca, verify-full}. Stored in credential storage. |
| SSL Client Key | `<textarea>` (credential) | `""` | n/a (credential) | **New.** Visible only when SSL Mode ∈ {verify-ca, verify-full}. Stored in credential storage. |

**Credential registration (D-02):**
```javascript
credentials: {
  user: { type: 'text' },
  password: { type: 'password' },
  sslCa: { type: 'text' },
  sslCert: { type: 'text' },
  sslKey: { type: 'text' },
}
```

#### Tab 3: Pool (`postgrestor-config-tab-pool`)

| Field | Widget | Default | typedInput Types | Notes |
|-------|--------|---------|-----------------|-------|
| Max Connections | typedInput | `10` | `num, flow, global` | Existing. Label: "Maximum size" |
| Idle Timeout | typedInput | `10000` (ms) | `num, flow, global` | Existing. Default changed from 1000→10000 (pg v8 default). Label: "Idle Timeout (ms)" |
| Connection Timeout | typedInput | `10000` (ms) | `num, flow, global` | Existing, unchanged. Label: "Connection Timeout (ms)" |
| Statement Timeout | typedInput | `0` (ms, 0 = no timeout) | `num, flow, global` | **New.** Pool-level default query timeout. Label: "Statement Timeout (ms)". Tooltip: "0 = no timeout. Applies as default to all queries through this pool." |

**Removed fields:**
- `min` (Minimum size) — pg.Pool v8 does not expose a functional `min` option; `min` has been deprecated in pg. Removed from Pool tab.

#### Pool Health Badge (POOL-02)

Rendered via `node.status()` at runtime on the PostgresDBNode. Displayed in the Node-RED runtime sidebar and optionally on the flow canvas.

| Property | Value |
|----------|-------|
| Update interval | 30 seconds (setInterval on `on('input', ...)`, cleared on `close`) |
| Source | `pool.totalCount`, `pool.idleCount`, `pool.waitingCount` |
| Format | `Active: {total - idle - waiting}  Idle: {idle}  Waiting: {waiting}  Total: {total}` |
| Healthy (active < 80% max) | Green ring |
| Near limit (active ≥ 80% max) | Yellow ring + `⚠` suffix |
| Error (pool in error state) | Red ring + error message |

## Interaction Contracts

### SSL Progressive Disclosure (D-01)

```
User changes SSL Mode dropdown:
  - 'disable' or 'require' → hide sslCa, sslCert, sslKey fields in Security tab
  - 'verify-ca' or 'verify-full' → show sslCa, sslCert, sslKey fields in Security tab
```

### DATABASE_URL Progressive Disclosure

```
User changes Database Source radio:
  - 'Manual' → show host, port, database fields; hide DATABASE_URL
  - 'DATABASE_URL' → hide host, port, database fields; show DATABASE_URL
```

### Deprecated SSL Migration (D-03)

```
On editprepare:
  - If config has ssl: true AND no sslmode set → sslmode = 'require'; show greyed-out SSL checkbox with tooltip
  - If config has sslmode set → hide deprecated SSL checkbox; use sslmode value
  - If config has ssl: false AND no sslmode → sslmode = 'disable'; hide SSL checkbox
```

### Named Parameter Binding (QUERY-01)

```
Interaction: User sends msg.params = { name: 'value', id: 42 }
Processing:
  1. Mustache renders query with msg context (if template tags present)
  2. msg.params keys sorted alphabetically, mapped to $1, $2, ...
  3. $1 = value of first alphabetical key, $2 = second, etc.
  4. Query executed with client.query(renderedQuery, positionalArray)
```

### Type Mapping Processing (REL-03)

```
Post-query processing, applied to each row:
  1. If mapNumeric AND column type OID === 1700 (numeric) → parseFloat(value)
  2. If mapTimestamptz AND column type OID ∈ {1184} → value.toISOString()
  3. If parseJsonb AND column type OID ∈ {3802} → JSON.parse(value), fallback to raw on parse error
  4. All toggles off → no mapping (existing behavior preserved)
```

### Prepared Statement Lifecycle (D-04, D-05, D-06)

```
No user interaction — fully transparent.

Runtime behavior:
  1. On first query execution after deploy: hash query text (SHA-256), generate name "ps_{hash[:8]}"
  2. Check per-connection cache: if name exists, use cached; if not, PREPARE name AS query
  3. Execute via EXECUTE name(params)
  4. On node close/remove: DEALLOCATE all prepared statements for this node's pool
  5. On redeploy: cache cleared; queries re-prepare on next execution
```
