## Locked Decisions (MUST implement exactly)

- **D-01:** Progressive disclosure — sslmode dropdown always visible; CA cert/client cert/client key fields appear only when sslmode is verify-ca or verify-full.
- **D-02:** Cert values in credential storage `type: 'password'` (encrypted). NOT plain typed-input fields.
- **D-03:** Deprecate boolean `ssl` toggle — existing `ssl: true` auto-migrates to `sslmode: require`. Deprecated field greyed out with migration tooltip.

## the agent's Discretion (resolved in this plan)

- **DATABASE_URL integration (POOL-04):** Radio toggle between Manual mode (host/port/database fields) and DATABASE_URL mode (single URL field). URL mode hides manual fields. Uses `pg-connection-string` `parse()` for URI decomposition. URL query params (sslmode, pool) augment config but editor fields override.
- **Pool health display (POOL-02):** Poll every 30 seconds via setInterval. Display "Active: N  Idle: M  Waiting: W  Total: T". Green ring when active < 80% max, yellow + ⚠ when ≥ 80% max, red on pool error. Pool error events set immediate red status.
- **SSL default idle timeout:** Bump idle default from 1000ms to 10000ms (pg v8 default per RESEARCH.md).
- **Remove `min` field:** pg.Pool v8 `min` is non-functional/deprecated. Remove from Pool tab + defaults (per UI-SPEC).
- **Pool-level statement_timeout:** Add `statementTimeout` field to Pool tab. Default 0 (no timeout). Applies as pool-level default; per-node timeout (QUERY-03 in plan 02-02) overrides it.

## Key Design Decisions

- **ACE editor for SQL:** RESEARCH.md confirms ACE is bundled with Node-RED 3.x and already in use. The UI-SPEC suggests CodeMirror 6 but RESEARCH identifies it as an unnecessary external dependency. We use ACE — bundled, proven, 0 install cost.
- **Type mapping defaults:** typeMapping toggles will be added to types.ts in this plan for completeness, but UI + runtime logic goes in plan 02-03.
</decisions>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Node-RED Editor → Runtime | SSL cert values transit from browser to server via Node-RED admin API; credentials are encrypted at rest |
| Node-RED Runtime → PostgreSQL | TLS connection established by pg.Pool; cert verification depends on sslmode |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | SSL certs in credential storage | mitigate | certs stored as `type: 'password'` in Node-RED credential storage (encrypted at rest); excluded from flow exports; D-02 explicitly requires this |
| T-02-02 | Spoofing | PostgreSQL server identity with sslmode:require | mitigate | `sslmode: require` encrypts but does NOT verify server identity; documented in tooltip; user must choose verify-ca/verify-full for full MITM protection; the plan includes all 4 sslmode options, including verify-ca and verify-full |
| T-02-03 | Tampering | Mustache template values → raw SQL | accept | Pre-existing from Phase 1 — Mustache interpolation renders values as raw SQL text, not parameterized; documentation warns against user-supplied data in templates; parameterized queries via `msg.params` (plan 02-02) provides the safe path |
| T-02-04 | Information Disclosure | DATABASE_URL in flow exports | mitigate | DATABASE_URL uses typedInput (str/flow/global/env); when set to `env` type, only the env var name is exported, not the value; users are guided via placeholder to use env vars for production |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed; `pg-connection-string` and `pg-types` are bundled with existing pg ^8.16.2 |
