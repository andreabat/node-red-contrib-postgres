## Decisions Made

- **RED runtime injection pattern:** Created `src/lib/red.ts` with `setRED()`/`getREDNodes()` to share the RED runtime across split TypeScript modules — avoids global variable pattern that doesn't work with ES module imports in separate files
- **ESLint relaxed rules:** Disabled `@typescript-eslint/no-explicit-any` and `@typescript-eslint/no-this-alias` because Node-RED patterns inherently use `any`-typed nodes and `const node = this` aliasing
- **Test strategy:** Used direct constructor testing with mocked pg.Pool and RED runtime rather than the full `node-red-node-test-helper` integration — simpler, faster, and more reliable for verifying the migration preserved behavior
- **Dual tsconfig approach:** Created `tsconfig.test.json` extending the main config because the main config excludes `__tests__/` from compilation, but ts-jest needs them included
