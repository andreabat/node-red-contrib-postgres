<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PostgreSQL → Listener node | NOTIFY payloads cross from external DB into Node-RED runtime — untrusted input |
| Listener channel config → PostgreSQL | Channel name interpolated into `LISTEN` SQL — SQL injection surface |
| Listener client lifecycle | Persistent pg client must be released on node close — connection leak risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Injection | LISTEN channel name | accept (documented) | Channel name interpolated directly into `LISTEN ${config.channel}` — no sanitization. Mitigation (pg-format identifier escaping) deferred to Phase 3 (LISTEN-02). Pre-migration behavior preserved. |
| T-01-10 | Injection | NOTIFY payload in msg | accept (preserved behavior) | Payload passed through as raw string in `msg.payload`. JSON parsing deferred to Phase 3 (LISTEN-03). Pre-migration behavior preserved. |
| T-01-11 | Denial of Service | Listener connection leak | mitigate | BUG-02 fix: `node.on('close')` performs `UNLISTEN` + `client.release()` with try/catch on both operations. Verified by Tests 4-5 — release guaranteed even on UNLISTEN failure. |
| T-01-12 | Denial of Service | Missing channel config | mitigate | Constructor validates `config.channel` — returns early with red status before attempting connection. Prevents pool.connect() call with undefined channel. |

**No blocking threats.** BUG-02 (connection leak) is the primary security improvement in this plan — fixed and tested. Accepted threats preserved from pre-migration.
</threat_model>
